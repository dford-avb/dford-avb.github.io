Overview

Redis is an open source, Berkeley Software Distribution (BSD) licensed, advanced key-value store that can optionally be used in Openmage for backend and session storage. In fact, you can replace memcached with Redis.

Following are some of the benefits Redis provides for Openmage implementations:

  • Redis supports on-disk save and master/slave replication. This is a powerful feature not supported by memcached. Replication enables high availability by eliminating a single point of failure.
  • Redis can be used for PHP session storage.
  • Redis provides much better eviction control and its backend is written with eviction support in mind.
  • Redis supports multiple databases that use the same server instance so you can use different databases for the Openmage cache, full page cache (EE only), and sessions without starting many processes listening on different ports.
  • Redis supports compression libraries gzip, lzf, and snappy. lzf and snappy are much faster than gzip.
  • Limits the number of concurrent lock requests before an HTTP 503 (Service Unavailable) error is returned.

Requirements for Using Redis with Openmage

The following Openmage editions support Redis session and backend caching:

  • Enterprise Edition (EE) 1.13 and later
  • Community Edition (CE) 1.8

The preceding Openmage editions support Redis server version 2.6.9 and later available from redis.io.

In addition, you can optionally use the Redis extension for PHP version 2.2.3 or later if you're using Redis for backend caching; however, Openmage works without this extension.

Configuring Redis

To use Redis with Openmage, you must configure Openmage to use Redis and you must install and configure the Redis server. These tasks are discussed in the following sections:

  • Configuring Openmage To Use Redis
  • Installing and Configuring the Redis Server

Configuring Openmage To Use Redis

To use Redis with Openmage, you only need to install and configure the Redis server. The integration between Redis and Openmage is already included with Openmage CE 1.8 and EE 1.13 and later versions. All you need to do is configure it.

Important: The Cm_RedisSession module in CE 1.8 is disabled by default. Openmage disables the module to avoid unnecessary connection tries to Redis when you choose to use file, database, or a different session storage method.

To enable Openmage to use Redis, perform the following tasks:

  1. Enable the Cm_RedisSession module.
    1. Open openmage-install-dir/app/etc/modules/Cm_RedisSession.xml in a text editor.
    2. Change the value of <active> to true.
    3. Save your changes to Cm_RedisSession.xml and exit the text editor.
  2. Modify openmage-install-dir/app/etc/local.xml.
    For configuration information, see the sample provided with Openmage in openmage-install-dir/app/etc/local.xml.additional and also see the Readme (session) and Readme (backend).
  3. Flush the Openmage cache in any of the following ways:
    • If you have access to the file system as the owner of the files in the Openmage installation directory, change to that directory and enter rm -rf var/cache.
    • Log in to the Admin Panel as an administrator. Click System > Cache Management, then click Flush Openmage Cache at the top of the page.

Installing and Configuring the Redis Server

Get Redis server version 2.6.9 or later and configure it according to their documentation.

You can optionally install the Redis extension for PHP version 2.2.3 or later as well, but Openmage functions without it.
