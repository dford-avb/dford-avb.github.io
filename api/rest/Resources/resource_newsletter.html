REST API: Newsletter

URI: /newsletter/subscribers

Allows you to retrieve subscribers.

URL Structure: http://openmagehost/api/rest/newsletter/subscribers
Version: 1

HTTP Method: GET /newsletter/subscribers

Description: Allows you to retrieve the list of existing subscribers.
Notes:: Only Admin user can retrieve the list of subscribers.

Authentication: Admin
Default Format: JSON
Parameters:
code

Example

GET http://openmagehost/api/rest/newsletter/subscribers?code=protest1

CUSTOM: adding parameter 

code
with a store code will retrieve list of customers for specific store.

Without store code, the customers endpoint will return the first 100 records. Using filters such as 

page
and 
limit
will allow to paginate through customers

Only pagination filters will work with this endpoint, Attribute filters will not work.

HTTP Method: POST /newsletter

Description: Allows you to create a new subscriber.
Authentication: Admin
Default Format: JSON
Parameters:

Name Description Required Type Example Value
first_name The customer first name optional string John
last_name The subscriber last name optional string Doe
email The subscriber email address required string johny@example.com
store_code AVBID associated with merchant required string protest1
external_subscriber_id subscriber id associated with third party app optional int 1

Example:

POST http://openmagehost/api/rest/newsletter

Request Body:

            {
                "email": "john@example.net",
                "store_code": "protest1",
                "first_name": "John",
                "last_name": "Doe",
                "external_subscriber_id": 1
            }

Response:
If the subscriber was created successfully, we receive Response HTTP Code = 201, and a Response body with the subscriber information.

           {
                "email": "testesataestsd@example.net",
                "first_name": "test",
                "last_name": "test last",
                "store_code": "protest1",
                "customer_id": 0,
                "message": "The subscription has been saved.",
                "status": "subscribed",
                "external_subscriber_id": 2,
                "external_sync_status": 1,
                "avb_subscriber_id": "561505"
            }

Note*: if external_subscriber_id is supplied, external_sync_status will automatically be applied, as the subscriber information is being provided through the external source.

Note**: avb_subscriber_id is a primary auto-increment key

Note***: If subscriber is existing customer, customer_id will automatically be linked from customer table

HTTP Method: PUT /newsletter

Description: Allows you to update an existing subscriber.
Authentication: Admin
Default Format: JSON
Parameters:

Name Description Required Type Example Value
first_name The customer first name optional string John
last_name The subscriber last name optional string Doe
email The subscriber email address required string johny@example.com
store_code AVBID associated with merchant required string protest1
is_subscribed change status of subscriber optional int 1
external_subscriber_id subscriber id associated with third party app optional int 1

Example:

POST http://openmagehost/api/rest/newsletter

Request Body:

            {
                "email": "john@example.net",

If the subscriber was updated successfully, we receive Response HTTP Code = 201, and a Response body with the subscriber information.

            }

Note*: if external_subscriber_id is supplied, external_sync_status will automatically be applied, as the subscriber information is being provided through the external source.

Note**: avb_subscriber_id is a primary auto-increment key

Note***: If subscriber is existing customer, customer_id will automatically be linked from customer table

REST API: Customer

URI: /customers/:customerId

Allows you to manage existing customers.

URL Structure: http://openmagehost/api/rest/customers/:customerId
Version: 1

HTTP Method: GET /customers/:customerId

Description: Allows you to retrieve information on an existing customer.
Notes:: The list of attributes that will be returned for customers is configured in the Openmage Admin Panel. The Customer user type has access only to his/her own information. Also, Admin can add additional non-system customer attributes by selecting Customers > Attributes > Manage Customer Attributes. If these attributes are set as visible on frontend, they will be returned in the response. Also, custom attributes will be returned in the response only after the customer information is updated in the Openmage Admin Panel or the specified custom attribute is updated via API (see the PUT method below). Please note that managing customer attributes is available only in Openmage Enterprise Edition.

Authentication: Admin, Customer
Default Format: XML
Parameters:
No Parameters

Example:

GET http://openmagehost/api/rest/customers/2

Response Body:

<?xml version="1.0"?>
<magento_api>
  <entity_id>2</entity_id>
  <website_id>1</website_id>
  <email>test@example.com</email>
  <group_id>1</group_id>
  <created_at>2012-03-22 14:15:54</created_at>
  <disable_auto_group_change>1</disable_auto_group_change>
  <created_in>Default Store View</created_in>
  <firstname>john</firstname>
  <lastname>Doe</lastname>
  <last_logged_in>2012-03-22 14:15:56</last_logged_in>
</magento_api>

response example: json

get http://openmagehost/api/rest/customers/141

response body:


{
  "entity_id": "2",
  "website_id": "1",
  "email": "test@example.com",
  "group_id": "1",
  "created_at": "2012-03-22 14:15:54",
  "disable_auto_group_change": "1",
  "created_in": "English",
  "firstname": "john",
  "lastname": "Doe"
}
HTTP Method: POST /customers/:customerId

Description: Not allowed.

HTTP Method: PUT /customers/:customerId

Description: Allows you to update an existing customer.
Notes: The list of attributes that will be updated for customer is configured in the Openmage Admin Panel. The Customer user type has access only to his/her own information.

Authentication: Admin, Customer
Default Format: XML
Parameters:
You must specify only those parameters which you want to update. Parameters that are not defined in the request body will preserve the previous values. The website_id and created_in attributes are not allowed for updating.

Example:

PUT http://openmagehost/api/rest/customers/2

Request Body:

<?xml version="1.0"?>
<magento_api>;
    <firstname>Earl</firstname>
    <lastname>Hickey</lastname>
    <email>customerss@example.com</email>
    <group_id>1</group_id>
</magento_api>
HTTP Method: DELETE /customers/:customerId

Description: Allows you to delete an existing customer.
Notes: Admin only can delete a customer.

Authentication: Admin
Default Format: XML
Parameters:
No Parameters

Example:

DELETE http://openmagehost/api/rest/customers/2
